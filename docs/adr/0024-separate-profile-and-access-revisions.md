# Separate profile and access revisions

Each environment profile persists a profile revision for every operation-affecting change and an access revision only for security-boundary changes. Endpoint, persisted API-generation policy, authentication strategy, principal, and secret advance both; timeout, proxy, and cache-policy changes advance only the profile revision; display name and suggested default namespace advance neither. An AUTO detection result belongs to a project session, enters its access identity, and advances only that session's epoch rather than either persisted revision. Open projects observing either operational revision capture a new context and session epoch, while only an access-revision change hides old caches and authentication sessions, avoiding needless cache loss for ordinary tuning.
