# Separate cache source, confirmation, and age

Every displayed dataset represents `DataSource` (REMOTE or CACHE), `DatasetConfirmation` (CONFIRMED, UNCONFIRMED, or REFRESH_FAILED), and, when cache-backed, `CacheAge` (WITHIN_TTL, STALE, or DEEP_STALE) as independent dimensions instead of collapsing them into one freshness enum. Restored cache is always UNCONFIRMED even within TTL; a write that may have left the client likewise invalidates confirmation of the retained pre-write detail until reconciliation, an allowed failed refresh changes confirmation without rewriting age, and elapsed time changes age without erasing failure evidence. Detail-load state, content-search coverage, summary-dataset completeness, and publish outcomes such as server-state-unknown remain separate again. The UI derives concise text and accessible tooltips from these dimensions without adding or replacing icon assets: list and detail continue to use status labels, while gutter markers retain the existing resolved, stale, and unresolved icons and use wording to distinguish unconfirmed from refresh-failed cached targets.
