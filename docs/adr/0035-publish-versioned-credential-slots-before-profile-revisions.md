# Publish versioned credential slots before profile revisions

PasswordSafe stores immutable credential slots keyed by profile ID and access revision rather than overwriting one active secret. A security-boundary edit stages the new secret under the unreachable next revision and publishes the corresponding profile and revisions only after that write succeeds, so a formal operation can capture either the complete old pair or the complete new pair but never an old access identity with new authority; both staging and profile publication recheck the profile lifecycle guard, and reverting to the same secret still creates a new revision. A failure before profile publication leaves the old pair active and the staged slot eligible for bounded idempotent orphan cleanup. If a persisted profile's required current slot is missing at startup, the profile enters `CredentialIncomplete`, hides its old-identity cache, and permits only repair instead of falling back to an older slot. Obsolete slots are removed asynchronously only after no capture can still depend on them; profile deletion persists its tombstone before deleting every slot so cleanup failures and late operations remain fail-closed.
