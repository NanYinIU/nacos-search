# Diagnose unsaved settings in isolation

Connection diagnostics run from an immutable snapshot of the unsaved settings form and sequentially validate local fields, resolve the API generation with the same candidate-probe algorithm but an isolated probe context and temporary authentication state, read one configuration-summary page from the explicitly configured namespace, and separately attempt namespace discovery. Success through the namespace read means the connection succeeds even when discovery is forbidden, and the report explicitly calls that outcome connected with manual namespace and unavailable discovery rather than connection failure. Diagnostics perform no writes and modify no persisted last-known generation, project session, cache, session epoch, shared request flight, or authentication registry. Their copyable report contains only stage status, timing, resolved generation, and sanitized failures, and applying settings causes the real project session to capture a fresh context.
