# Keep authentication strategies explicit

Nacos Search supports ANONYMOUS, NACOS_PASSWORD, HTTP_BASIC, and BEARER_TOKEN as explicit P0 authentication strategies and never falls back automatically between them. NACOS_PASSWORD delegates login to the selected v1 or v3 adapter and, on V3, is supported only when the server reports the built-in Nacos or LDAP authentication system; another auth-system type yields an explicit strategy/capability mismatch. HTTP_BASIC remains available for reverse proxies and custom deployments, while BEARER_TOKEN stores the externally issued secret in PasswordSafe and is likewise a proxy/custom-auth contract for V1 rather than a promised native Nacos 2.x capability. P1 may add OIDC client credentials and automatic refresh when demand is proven, while interactive browser SSO and arbitrary custom headers remain outside this scope.
