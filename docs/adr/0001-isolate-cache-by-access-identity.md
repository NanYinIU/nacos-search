# 按访问身份隔离缓存

Nacos Search 的缓存和已完成 Token 会话必须按环境档案 ID、访问修订号、规范化服务端地址、已解析 API 世代、认证策略和主体组成的访问身份隔离；匿名访问也是独立身份，密码与 Token 本身不得进入身份键。正式请求及登录/刷新 single-flight 还必须包含捕获的档案修订号或等价执行策略指纹，防止新 timeout/proxy policy 加入旧 flight。服务端、持久化 API 世代策略、认证策略、主体或密钥变化时递增访问修订号，使旧身份数据立即不可见并等待异步清理，即使密钥后来改回原值也不重新使用旧权限上下文；AUTO 探测结果已直接进入身份，不再改变修订号，展示名称、超时和搜索偏好不影响访问修订号。世代 probe 和隔离诊断尚无完整访问身份，必须使用专用非缓存键且不得加入正式 Token 或请求 single-flight。升级时无法证明访问身份归属的旧格式配置缓存一次性废弃，不猜测迁移；清理必须可中断、可重复，失败不能阻塞插件启动。
