# Recover authentication without changing strategy

Only NACOS_PASSWORD authentication recovers automatically when its selected adapter semantically maps a response to `AuthenticationFailed` caused by an expired or invalid login token: it invalidates the token for the captured access identity, logs in once with the operation's credential snapshot, and replays an idempotent read at most once. This decision is not keyed to a raw HTTP status because Nacos V1 may report an invalid token with 403; a response mapped to `PermissionDenied` is never refreshed. ANONYMOUS, HTTP_BASIC, and BEARER_TOKEN return explicit authentication failures, and writes with an authentication or ambiguous transport failure are never replayed automatically; no authentication outcome can change the selected API generation or authentication strategy.
