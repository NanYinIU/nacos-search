# Use operation-aware sanitized failures

Protocol adapters map responses into InvalidConfiguration, TransportFailure, EndpointRedirected, AuthenticationFailed, PermissionDenied, GenerationUnsupported, CapabilityUnsupported, ResourceNotFound, RateLimited, ServerFailure, MalformedResponse, WriteConflict, AmbiguousWriteResult, or Cancelled. `WriteConflict` represents an authoritative V1 CAS rejection or another protocol-level concurrent-modification result, while `AmbiguousWriteResult` means the server outcome is unknown. `TransportFailure` carries only a safe `TransportKind` such as CONNECT, CONNECT_TIMEOUT, READ_TIMEOUT, DNS, or TLS so the transport seam can apply one retry policy per kind; `EndpointRedirected` may carry a structured sanitized endpoint with user-info, query, and fragment removed. Adapters interpret both operation context and protocol semantics—including HTTP status plus the V3 JSON code—before choosing a failure; a 404 alone is therefore not always `GenerationUnsupported`. Failures otherwise retain only operation, status, request ID, and a sanitized bounded message, never raw bodies, secrets, authentication headers, tokenized URLs, or configuration content.
