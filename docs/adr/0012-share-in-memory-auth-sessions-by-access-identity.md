# Share in-memory auth sessions by access identity

NACOS_PASSWORD tokens live only in an application-level authentication-session registry keyed by the complete access identity, allowing projects with the same identity to share a token without crossing a security revision. The completed token remains identity-keyed, but each in-flight login or refresh is single-flight by identity plus captured profile revision or equivalent execution-policy fingerprint, so a new timeout/proxy configuration cannot join an old flight. A failed proactive refresh may keep using an unexpired token but an expired token fails explicitly, invalidation targets one identity, and closing one project does not log out another; external BEARER_TOKEN credentials remain operation snapshots, while future OIDC refresh can reuse the registry abstraction.
