# Bootstrap AUTO generation before constructing an access identity

AUTO detection captures an immutable `GenerationProbeContext` before a complete access identity exists and first invokes the candidate V3 state probe with the operation's credential snapshot; a candidate that needs NACOS_PASSWORD login may create an isolated temporary token, but that token never enters the shared authentication registry. Only typed `GenerationUnsupported` from V3 followed by a positively successful candidate V1 state probe resolves V1; `CapabilityUnsupported`, a raw 404, authentication, authorization, timeout, redirect, malformed, rate-limit, and server failures stop detection without fallback. Before each consumer commits a shared probe result to its session or persists `lastKnownGeneration`, it independently rechecks captured profile ID, profile revision, access revision, session epoch, and absence of a profile tombstone. Successful formal detection assigns the result to the requesting project session, advances its epoch when the result changes, and persists `lastKnownGeneration` by profile ID, access revision, and canonical endpoint solely to locate the same identity's unconfirmed cache during a later offline restart. The last-known value never skips formal detection, authorizes a remote operation, or advances a profile revision; a newly resolved different generation immediately switches identity and hides the old cache. When formal detection fails, only transient transport, rate-limit, or server failures may use a matching last-known value to display cache as `REFRESH_FAILED` without remote operations; authentication or permission failure blocks that cache, while invalid configuration, redirect, malformed response, or generation failure exposes none. Without a matching last-known value AUTO exposes no cache. Concurrent formal probes may share only a safe probe flight keyed by the complete non-secret probe inputs, while isolated diagnostics share neither that flight nor temporary authentication state.
