# Simplify settings with progressive disclosure

Most users only need a successful connection, so the environment-profile settings primary form keeps display name, endpoint, username, password, namespace, and test-connection, while authentication strategy, API generation policy, write intent, and other tuning move under Advanced. The Advanced authentication control exposes only the P0 strategies (ANONYMOUS, NACOS_PASSWORD, HTTP_BASIC, BEARER_TOKEN) with plain-language labels and short tooltips; legacy BASIC, TOKEN, and HYBRID remain migration-only and never appear in the chooser. API generation policy stays default AUTO (ADR 0006) and is labeled “API 版本” in the UI while domain language remains API 世代. Credential-driven inference never cross-switches strategies at runtime (ADR 0007): new empty-credential profiles start as ANONYMOUS, the default family may only auto-upgrade ANONYMOUS to NACOS_PASSWORD when both username and password become complete, HTTP_BASIC and BEARER_TOKEN stay locked against primary-form inference, choosing ANONYMOUS disables and clears credentials on apply, and BEARER_TOKEN relabels the secret field as Token.
