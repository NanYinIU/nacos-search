# Prefer v3 and fall back only on definitive absence

Each environment profile defaults to the persisted AUTO API-generation policy, probes the public read-only V3 state endpoint first, and fixes the detected generation only in the project session and every derived operation context. AUTO may start a candidate V1 probe only after the V3 adapter returns the typed `GenerationUnsupported`, never for `CapabilityUnsupported`, a raw 404, authentication, authorization, timeout, redirect, proxy, or server failures, and selects V1 only when that candidate probe positively succeeds. Users can force V1 or V3 for unusual deployments. Applying an operation-affecting profile change, explicitly reconnecting, or restarting the IDE makes the real project session detect again, while an isolated diagnostic performs and reports its own detection without changing that session or either persisted revision.
