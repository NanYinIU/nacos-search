# Gate releases with contract and live Nacos matrices

Every change runs V1 and V3 adapter contracts plus AUTO failure classification, offline bootstrap, credential crash-window, authentication execution-flight, cache high-water, tombstone late-completion, and ambiguous-write reconciliation tests, with live smoke coverage for Nacos 2.5.3 and a standard Nacos 3.2.3 without the legacy adapter. V1 paging/search and state-probe shapes are pinned implementation contracts because the public V1 OpenAPI manual does not specify them, and the `casMd5` HTTP header is pinned to official 2.5.3 server source plus live behavior rather than advertised as a public HTTP contract. Nightly and release validation expands to 2.0.4, 2.2.4, 2.5.3, 3.0.3, 3.1.2, and 3.2.3 using two explicit server configurations: authentication disabled for anonymous coverage, and an initialized Nacos username/password for authenticated coverage rather than assuming a distribution default password. Manual namespace without discovery, read-only publish denial, metadata-preserving verified write, and V1 CAS conflict scenarios are mandatory; LDAP integration is fixed to 3.2.x, while Basic and Bearer retain protocol contracts. Any cross-identity cache exposure, misdirected or replayed write, tombstone resurrection, or erroneous V3-to-V1 fallback blocks P0.
