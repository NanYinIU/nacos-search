# Migrate global selection into project workspaces

Existing global environment profiles retain stable IDs and migrate complete PasswordSafe credentials into revision-keyed slots before publishing the new profile state, while the former global activeServerId becomes only the default profile for newly initialized projects and each profile's namespace becomes only its suggested default namespace. A project without session state initializes once from those defaults and thereafter stores its selected profile, namespace, search, and UI state only in non-shareable workspace storage; global recent-search and pagination state are discarded, legacy flat fields remain deserialization-only for one release, and the versioned migration is idempotent and assigns any missing profile ID exactly once. A missing or half-populated required credential and an endpoint rejected by canonical validation become visible `ConfigurationRequired` repair states rather than falling back to another slot or appearing as empty results; completed selection moves and discarded UI state are listed once in the project's dismissible upgrade summary.
