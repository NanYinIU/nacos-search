# Show migration and degraded-discovery states explicitly

Local defects that prevent a safe operation context, including `CredentialIncomplete` and a legacy endpoint rejected by canonical validation, put the selected profile in blocking `ConfigurationRequired`: no request is sent, old-identity cache remains hidden, and the tool window offers a direct settings repair action instead of showing an empty result. Completed migration effects—retired legacy cache, one-time project selection initialization, and discarded global search or pagination state—appear once per project in a dismissible non-modal `UpgradeCompleted` summary and never trigger a startup dialog. A successful configured-namespace read with failed or forbidden namespace discovery is separately shown as connected with manual namespace and discovery unavailable. These states reuse existing text banners, status labels, and warning styling without requiring new icon assets.
