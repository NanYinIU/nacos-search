# Fence project-session results with an epoch

Each project session owns an in-memory epoch that advances when its selected environment, namespace, profile revision, access revision, or resolved API generation changes. A switch attempts to cancel obsolete foreground work, but correctness depends on checking the captured epoch before publishing UI state or scheduling follow-up work: an obsolete operation may finish and populate only the cache coordinate derived from its captured access identity and namespace, never one recomputed from current settings, and a shared request remains alive while another project still consumes it. This permission ends when a persisted profile-deletion tombstone causes the cache seam to reject every mutation from that profile, regardless of the captured epoch or remaining consumers.
