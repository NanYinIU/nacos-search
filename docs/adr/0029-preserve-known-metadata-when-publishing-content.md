# Preserve known metadata when publishing content

P0 edits configuration content only: ConfigDetail retains every field an adapter can read, and PublishCommand derives from the just-refetched remote detail, replacing only content while preserving the exact remote type and other known metadata. As a product safety policy, the v3 adapter resends known `appName`, `desc`, `configTags`, `type`, and `encryptedDataKey` values even though the protocol marks them optional; the v1 adapter sends only fields in its stable contract. Publishing is blocked when known metadata cannot be preserved, read-back verifies every preservable field the adapter can retrieve using field-specific semantic equality, and a non-empty encrypted-data key makes the configuration read-only to avoid corrupting encrypted content.
