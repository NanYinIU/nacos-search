# Bind edit sessions to their original target

Starting an edit creates an in-memory session bound to the profile ID, access identity, canonical namespace, data ID, group, and base MD5 and content, but never to a password or token; publishing derives its target only from that binding and captures fresh operation credentials. A dirty edit blocks selecting another configuration, switching environment or namespace, disabling writes, deleting its profile, closing the project, or destroying the tool-window content until the user cancels the action or explicitly discards the draft; P0 provides no draft shelf and never persists configuration content. `PUBLISHING` and `VERIFYING` forbid those target-destroying actions rather than offering ordinary discard, and `SERVER_STATE_UNKNOWN` permits only reconciliation, copying the draft, or an explicit abandon action that warns the server may already have changed and never updates cache. An operation-affecting profile change retains the draft but requires a fresh preflight, while an access-revision or resolved-generation change invalidates publishing and leaves the text available only for copying.
