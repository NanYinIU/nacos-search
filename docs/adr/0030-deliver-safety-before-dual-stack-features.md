# Deliver safety foundations before dual-stack features

P0 is implemented in dependency order: P0-A introduces global profiles, project sessions, dual revisions, immutable contexts, versioned credential slots and snapshots, the minimal candidate-probe and adapter registry required to construct identities, authentication sessions, unified transport and failures, and the new cache schema; P0-B completes V1 and V3 read parity, API selection, authentication, manual namespace fallback, metadata-first search and detail flows, PSI, and isolated diagnostics; P0-C restores controlled publishing behind per-profile opt-in with bound edit sessions, observed-conflict preflight, one non-retried write, read-back verification, and metadata preservation, while V3 remains publish-capable without a CAS guarantee. P1 then gives each adapter same-identity read-only history with historical-version IntelliJ diff, removes the now-unused one-release legacy migration and cleanup code after its compatibility window, and may add OIDC client-credential refresh when demand is proven. Interactive SSO, Console API fallback, full content scans, history restore, deletion, gray release, cross-environment promotion, and custom TLS or certificate-management behavior remain uncommitted.
