# Do not fall back when a shared profile is deleted

Deleting a shared environment profile warns about its open-project consumers and is refused while any consumer owns a dirty edit until that user explicitly discards the draft. After confirmation, deletion first persists a profile tombstone that prevents new context capture and becomes a common lifecycle guard for profile/credential publication, authentication-token publication, last-known-generation updates, probe/session commits, and every cache mutation; each asynchronous completion rechecks the guard before publishing. Only then does deletion move referring sessions to ProfileUnavailable, advance their epochs, and cancel or detach obsolete foreground work instead of selecting another environment. PasswordSafe slots, exact authentication sessions, and caches are subsequently removed, the new-project default is cleared when necessary, and closed projects await an explicit profile choice on their next open. Cleanup may be retried, but no late shared flight can cross the tombstone or silently target another cluster.
