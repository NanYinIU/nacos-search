# Model server origin and context path separately

An environment profile models its Nacos endpoint as an origin containing only scheme, host, and port plus an explicit context path that defaults to `/nacos` and may be `/` or a reverse-proxy path. Canonicalization accepts only absolute HTTP/HTTPS URIs, lowercases scheme and the IDNA ASCII host without DNS resolution, removes the scheme's default port, preserves an explicit non-default port, and renders IPv6 as a bracketed lowercase literal. The context path gains one leading slash and loses a trailing slash except for `/`; ambiguous empty segments, dot segments, backslashes, control characters, and encoded slash or backslash are rejected, percent hex is uppercased, and unreserved characters are decoded. Protocol adapters append only relative paths, legacy URLs without a path migrate to `/nacos` while valid existing paths are preserved, user-info, query, and fragment are rejected, no alternate context path is probed automatically, and changing either component advances both profile and access revisions and changes the canonical endpoint in the access identity.
