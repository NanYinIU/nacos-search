标注在 @NacosValue / @Value 占位符行旁的 12×12 代码槽图标。点击跳转到对应 Nacos 配置。与工具窗口图标同一视觉家族，但收敛到代码槽尺寸——去掉文档外框与折角（紧贴的代码已说明「这是配置」），并把配置 key/value 收成一笔连体的字形：左侧 [ 形的 key 行 + 左杆 + value 行，末端在正中直接拐成斜向「跳转」箭头杆——配置即出口。状态差异只落在箭头头上。

状态语义

状态触发条件（来自 LineMarkerProvider）视觉差异
正常态
 dataId != null 或 NacosKeyResolver.hasKey() 命中——配置已在缓存中，点击直达详情。 箭头实心，描边强灰 #6C707E / #CED0D6
置灰态
 键未命中缓存，需走 lazyLoadAndNavigate() 远程兜底——可点击，但未必能定位到详情。 箭头空心，描边弱灰 #9AA0A8 / #787C85

接入（3 步）

1 · 把四个 SVG 放进 src/main/resources/icons/
# 正常态
nacosConfigGutter.svg
nacosConfigGutter_dark.svg
# 置灰态
nacosConfigGutterUnresolved.svg
nacosConfigGutterUnresolved_dark.svg
2 · NacosIcons.kt — 注册两态图标
object NacosIcons {
    val ToolWindow = IconLoader.getIcon("/icons/nacosSearch_20.svg", NacosIcons::class.java)
    // gutter 跳转标记 · 双态
    val GutterConfig = IconLoader.getIcon("/icons/nacosConfigGutter.svg", NacosIcons::class.java)
    val GutterConfigUnresolved = IconLoader.getIcon("/icons/nacosConfigGutterUnresolved.svg", NacosIcons::class.java)
}
3 · NacosValueLineMarkerProvider.kt — 按解析状态选图标
// markerForLiteral(...) 内，原先固定用 NacosIcons.ToolWindow：
val resolvable = codeContext.dataId != null ||
    NacosKeyResolver.hasKey(placeholder.key, NacosKeyResolver.currentServerUrl())
val icon = if (resolvable) NacosIcons.GutterConfig else NacosIcons.GutterConfigUnresolved

return LineMarkerInfo(
    anchor, anchor.textRange, icon,
    { if (resolvable) "Nacos 配置：跳转到详情" else "Nacos 配置：尝试远程定位" },
    { _, _ -> navigateFromCode(anchor, literal, placeholder.key, codeContext) },
    GutterIconRenderer.Alignment.RIGHT,
    { "nacos.value.${placeholder.key}" }
)

提示：shouldShowMarker() 现在仍只决定「是否显示」；上面把同一个判定复用为「显示哪一态」。IDE 在暗色主题下会自动选用 _dark 文件，无需在代码里区分主题。
