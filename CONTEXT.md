# Nacos Search

Nacos Search 在 IDE 内连接 Nacos 环境、查找配置并呈现配置数据。本词汇表统一描述搜索结果及其可信状态的领域语言。

## Language

**展示数据集**：
工具窗口当前向用户呈现的一组配置数据。
_Avoid_: 当前缓存、当前结果

**数据来源**：
展示数据集的取得途径，取值为远程响应或本地缓存；它不表示数据是否仍可确认有效。
_Avoid_: 数据状态、新鲜度

**数据集新鲜度**：
插件对展示数据集与当前 Nacos 服务端状态一致程度的判断，取值为新鲜、陈旧或未知。
_Avoid_: 缓存状态、数据来源

**新鲜**：
展示数据集来自最近一次成功请求，或来自仍在有效期内且未发生后续刷新失败的缓存。
_Avoid_: 实时

**陈旧**：
展示数据集已过有效期，或用户要求刷新但插件未能确认服务端当前状态。
_Avoid_: 离线数据、旧缓存

**未知**：
插件恢复了持久化数据，但本次运行尚未成功确认当前 Nacos 环境状态。
_Avoid_: 未加载、陈旧

**访问身份**：
用户以特定环境配置、认证模式和用户名访问 Nacos 的身份；匿名访问也是独立身份。
_Avoid_: 账号、凭据、Token

**已解析配置引用**：
插件已经确认在当前访问身份和 namespace 中存在对应配置的代码引用。
_Avoid_: 可用引用、缓存命中

**未解析配置引用**：
插件已经成功检查当前访问身份和 namespace，并确认对应配置不存在的代码引用。
_Avoid_: 无法解析、不可用引用

**不可判定配置引用**：
插件因数据未就绪、网络或认证问题、或操作取消而无法确认配置是否存在的代码引用。
_Avoid_: 未解析配置引用、缺失配置

**缓存导航目标**：
由本地缓存提供、可供离线查看但不证明配置当前仍存在于 Nacos 的导航目标。
_Avoid_: 已解析配置引用、实时配置

**深度陈旧阈值**：
区分普通陈旧数据与必须强制单项刷新的时间阈值，固定为七天；它不是导航或缓存保留上限。
_Avoid_: 最大陈旧期、缓存有效期、TTL

**Namespace 索引**：
特定访问身份与 namespace 下用于全局搜索和配置存在性判断的完整配置集合。
_Avoid_: 配置详情缓存、列表页缓存

**配置详情缓存**：
特定访问身份、namespace、Data ID 与 Group 对应的单个配置内容副本。
_Avoid_: Namespace 索引

**列表页缓存**：
特定访问身份、namespace、搜索条件和页码对应的配置元数据页面。
_Avoid_: Namespace 索引、配置详情缓存


**数据集完整性**：
插件对预期配置是否全部取得的判断，取值为完整、部分或失败；它不表示数据是否新鲜。
_Avoid_: 数据集新鲜度、请求状态

**部分数据集**：
可靠列表已经取得，但一个或多个配置详情未能取得的数据集。
_Avoid_: 不完整缓存、失败结果

**诊断快照**：
用户发起连接诊断时从当前设置表单取得、仅供本次诊断使用且不改变插件活动环境的连接信息。
_Avoid_: 临时环境、未保存配置
