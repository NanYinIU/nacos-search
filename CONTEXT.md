# Nacos Search

Nacos Search 在 IDE 内连接 Nacos 环境、查找配置并呈现配置数据。本词汇表统一描述搜索结果及其可信状态的领域语言。

## Language

**展示数据集**：
工具窗口当前向用户呈现的一组配置数据。
_Avoid_: 当前缓存、当前结果

**数据来源**：
展示数据集的取得途径，取值为远程响应或本地缓存；它不表示数据是否仍可确认有效。
_Avoid_: 数据状态、新鲜度

**数据集确认状态**：
插件在本次运行中对展示数据集与当前 Nacos 服务端关系的判断，取值为已确认、未经确认或刷新失败；它与缓存年龄彼此独立。
_Avoid_: 数据来源、数据集新鲜度、缓存年龄

**已确认**：
本次运行中的成功远程观测已经确认该数据集；仅处于 TTL 内不能使恢复的缓存成为已确认。
_Avoid_: 新鲜、实时

**未经确认**：
当前展示数据在本次运行中尚无成功远程观测确认，或一次可能已经发出的写入使此前确认失效；典型情况包括恢复同一访问身份的持久化数据，以及结果不确定时暂时保留的写前详情。
_Avoid_: 未加载、陈旧、刷新失败

**刷新失败**：
用户或后台已经尝试刷新，但允许保留旧数据的类型化失败使本次运行无法确认服务端当前状态；失败类型与作用域仍独立保留。
_Avoid_: 陈旧、离线、请求失败

**缓存年龄**：
缓存条目相对 TTL 与七天深度陈旧阈值的时间状态，取值为 TTL 内、陈旧或深度陈旧；年龄变化不覆盖数据集确认状态。
_Avoid_: 数据集确认状态、数据来源、缓存状态

**TTL 内**：
缓存条目尚未超过配置的有效期；它不表示本次运行已经远程确认。
_Avoid_: 已确认、新鲜

**陈旧**：
缓存条目已经超过 TTL，但尚未超过七天深度陈旧阈值。
_Avoid_: 刷新失败、未经确认、离线数据

**深度陈旧**：
缓存条目已经超过固定七天阈值；它仍可作为缓存导航目标，但打开时必须触发单项刷新。
_Avoid_: 已删除、不可导航、缓存保留上限

**访问身份**：
用户通过特定环境档案、访问修订号、服务端地址、已解析 API 世代、认证策略和主体访问 Nacos 的安全边界；匿名访问也是独立身份，密钥不属于身份键。
_Avoid_: 账号、凭据、Token

**访问修订号**：
环境档案中持久化的单调递增编号；服务端地址、API 世代策略、认证策略、主体或密钥变化时递增，使旧权限上下文的缓存与认证会话立即不可见；AUTO 的探测结果不改变该编号。
_Avoid_: 配置版本、缓存版本、密钥哈希

**档案修订号**：
环境档案中持久化的单调递增编号；任何影响后续操作行为的设置变化时递增，用于通知项目会话重新捕获上下文，但不必改变缓存身份。
_Avoid_: 访问修订号、会话纪元、设置 schema 版本

**环境档案**：
可供多个项目复用的一组 Nacos 连接与认证设置；它本身不表示任何项目当前正在访问该环境。
_Avoid_: 当前环境、项目会话

**服务端端点**：
由仅含 scheme、host、port 的 origin 与显式 context path 组成的规范 Nacos 应用根地址；协议适配器只追加相对路径。
_Avoid_: 请求 URL、服务器地址字符串、Console 地址

**项目会话**：
单个 IDE 项目当前选择的环境档案与 namespace；不同项目的项目会话彼此独立。
_Avoid_: 全局环境、环境档案

**档案不可用**：
项目会话引用的环境档案已被删除时的阻断状态；项目不会自动切换环境，必须由用户重新选择。
_Avoid_: 连接失败、未配置、默认环境

**档案删除墓碑**：
共享环境档案确认删除后首先持久化的不可逆屏障；它禁止该档案产生新操作上下文或缓存 mutation，并使删除前的晚到响应无法复活凭据、认证会话或缓存。
_Avoid_: 档案不可用、删除确认、缓存清理任务

**会话纪元**：
项目会话中仅驻内存的单调递增编号；环境、namespace、档案修订号、访问修订号或已解析 API 世代变化时递增，用于阻止旧操作更新新的会话状态。
_Avoid_: 访问修订号、请求编号、缓存版本

**操作上下文**：
一次操作开始时从项目会话捕获的不可变访问目标，包含访问身份、namespace 与该操作适用的连接策略；之后的会话切换不改变它。
_Avoid_: 当前设置、活动环境、临时环境

**执行键**：
正式请求或登录/刷新 flight 的去重边界，由访问身份、档案修订号或等价执行策略指纹、操作及规范参数组成；缓存和已完成 Token 仍只按访问身份隔离。
_Avoid_: 访问身份、缓存键、观测序号

**写入开关**：
环境档案中表达用户是否允许插件发起普通配置发布的显式设置；默认关闭，开启不表示服务端已经授予写权限。
_Avoid_: 写权限、认证策略、编辑状态

**编辑会话**：
用户开始修改配置时创建的内存态草稿，固定绑定访问身份、Namespace、配置坐标与基线版本，但不持有凭据；当前 UI 选择不能改变其发布目标。
_Avoid_: 操作上下文、项目会话、配置详情缓存

**已验证发布**：
发布前观察远端等于编辑基线、用户确认目标与差异、发布后读回正文及应保留元数据在适配器语义上等于发布命令的写入结果；只有该结果可以清除 dirty 状态，并把发布命令的结果作为当前详情写入缓存。任何 reconciliation 读回仍是独立的远程观测，按 observation high-water 刷新、删除或阻断缓存。对无 CAS 的 V3，该内部状态只确认读回当时的服务端结果，不保证 preflight 与 write 之间没有并发覆盖，也不是新的用户可见发布类型。
_Avoid_: 发布成功、HTTP 2xx、已保存

**发布命令**：
从发布前最新远端详情派生、仅用草稿替换正文并保留适配器已知元数据的单次写入意图；它固定绑定编辑会话目标。
_Avoid_: 编辑草稿、HTTP 请求、配置详情

**Namespace 发现**：
插件为项目会话枚举可选 namespace 的能力；发现失败或无权限不表示连接失败，也不表示手工指定的 namespace 不可访问。
_Avoid_: 连接检查、Namespace 访问权限

**发现降级连接**：
目标 Namespace 的配置读取已经成功、但 Namespace 发现不可用时的连接状态；插件继续使用手工 Namespace，并将发现限制作为独立警告呈现。
_Avoid_: 连接失败、空 Namespace、匿名访问

**Namespace 标识**：
请求、缓存与会话使用的规范化 namespace ID；空值、空串和精确的 public 统一表示为 public，展示名称不属于该标识。
_Avoid_: Namespace 名称、tenant 参数、Namespace 选项

**API 世代**：
一个操作上下文使用的 Nacos 协议契约，取值为 V1 或 V3；它不等同于完整的服务端版本号。
_Avoid_: Nacos 版本、认证模式、AUTO

**API 世代策略**：
环境档案持久化的协议选择意图，取值为 AUTO、V1 或 V3；修改它会推进档案修订号与访问修订号，AUTO 的已解析结果则只属于项目会话。
_Avoid_: API 世代、服务端版本、世代探测结果

**世代探测**：
项目会话在 AUTO 策略下以无副作用请求确定 API 世代的过程；只有确认目标世代不受支持时才允许回退。
_Avoid_: 连接诊断、认证回退、版本猜测

**世代探测上下文**：
AUTO 探测开始时捕获的不可变安全输入；它在完整访问身份形成之前临时选择候选适配器，可配合独立凭据快照，但不进入正式缓存、Token 会话或请求 single-flight。
_Avoid_: 操作上下文、访问身份、诊断快照

**最近已知世代**：
按环境档案 ID、访问修订号和规范端点保存的上次成功 AUTO 探测结果；它只允许离线重启时定位同一旧身份的缓存并标为未经确认，不能跳过正式探测或授权网络操作。
_Avoid_: API 世代策略、当前 API 世代、离线自动探测

**协议适配器**：
由已解析 API 世代唯一选择、负责端点、参数、响应、认证传递和错误映射的 V1 或 V3 实现；上层业务不判断协议世代。
_Avoid_: API 客户端、认证策略、能力探测

**协议能力**：
协议适配器显式声明的可用操作，例如 Namespace 发现或历史查询；认证、授权和网络失败不能被解释为能力不支持。
_Avoid_: 权限、功能开关、版本字符串

**只读配置历史**：
当前访问身份、Namespace 与配置坐标内由对应协议适配器取得的历史版本列表和正文；V1 与 V3 提供能力对等而不相互混合，且不包含恢复、回滚或重新发布。
_Avoid_: 跨世代历史、审计日志、配置备份

**历史差异**：
两个同身份历史版本之间或历史版本与当前详情之间的 IntelliJ Diff 展示；它不创建发布命令，也不改变编辑会话。
_Avoid_: 发布确认差异、回滚预览、跨环境比较

**请求失败**：
协议适配器结合具体操作解释并清洗后的类型化失败；它包含统一重试语义，包括独立的写入冲突，但不携带原始响应体、认证信息或配置正文。
_Avoid_: HTTP 状态码、异常字符串、连接状态

**请求预算**：
一次网络交换及其传输重试和认证重放共同消耗的总时限；重试或重登录不能重置该时限。
_Avoid_: 连接超时、读取超时、操作超时

**结果不确定**：
写请求可能已经发出后因连接中断、取消等原因无法确认服务端是否已应用变更的发布结果状态；它禁止自动重放和再次发布，保留草稿，并要求使用原操作上下文重新读取服务端状态。它不属于展示数据集的确认状态，也不把旧缓存变成第四种 `DatasetConfirmation`；但写前详情此时不再保持已确认，而是先变为未经确认，读回失败时再按类型化失败矩阵变为刷新失败或被相应作用域阻断。
_Avoid_: 发布失败、超时、可重试错误、数据集确认状态

**访问受阻**：
适配器确认认证失败时针对访问身份、确认配置读取无权限时针对访问身份与 Namespace 持久化的缓存可见性状态；写入、Namespace 发现和其他能力的权限失败只影响对应能力。
_Avoid_: 认证失败、离线、缓存失效

**认证策略**：
环境档案显式选择的身份获取与传递方式，P0 取值为 ANONYMOUS、NACOS_PASSWORD、HTTP_BASIC 或 BEARER_TOKEN；API 世代只决定该策略使用的协议细节，不改变认证策略本身。
_Avoid_: 认证模式、API 世代、自动认证

**凭据快照**：
一次操作开始时读取并固定的临时认证材料；正式操作从 PasswordSafe 取得，隔离诊断从未保存表单取得，它只驻留内存，不属于访问身份，不参与相等性、日志、持久化或缓存键。
_Avoid_: 操作上下文、访问身份、凭据引用

**凭据槽**：
PasswordSafe 中按环境档案 ID 与访问修订号定位的不可变密钥版本；profile 只有发布对应 revision 后，新槽才可被正式操作读取，旧槽不能作为缺失凭据的回退。
_Avoid_: 凭据快照、当前密码、Token 会话

**凭据不完整**：
当前认证策略需要密钥但其 revision 对应凭据槽不存在或旧设置只迁移出半套凭据时的 fail-closed 档案状态；它阻止远程操作并要求用户修复，不会尝试旧槽或另一认证策略。
_Avoid_: 认证失败、匿名访问、档案不可用

**需要配置**：
环境档案因凭据不完整、端点不合法等本地问题无法安全构造操作上下文时的阻断状态；它隐藏旧身份缓存、禁止网络请求，并提供修复设置入口。
_Avoid_: 空结果、连接失败、档案不可用

**升级完成摘要**：
新版本首次完成项目状态迁移后展示一次且可关闭的非阻塞说明，列出旧缓存废弃、项目选择初始化以及旧搜索或分页状态丢弃等已完成动作。
_Avoid_: 错误弹窗、需要配置、迁移日志

**认证恢复**：
NACOS_PASSWORD 操作收到适配器确认的 Token 失效或过期错误后，仅失效当前访问身份的 Token，使用原凭据快照重新登录，并至多重放一次幂等读操作的过程；它不由裸 HTTP 状态码决定。
_Avoid_: 请求重试、API 回退、权限降级

**认证会话**：
应用内按访问身份隔离并可被多个项目复用的临时 Nacos 登录状态；它只驻留内存，与项目会话和外部 Bearer 凭据无关。
_Avoid_: 项目会话、凭据快照、持久化登录

**已解析配置引用**：
插件已经确认在当前访问身份和 namespace 中存在对应配置的代码引用。
_Avoid_: 可用引用、缓存命中

**未解析配置引用**：
插件已经成功检查当前访问身份和 namespace，并确认对应配置不存在的代码引用。
_Avoid_: 无法解析、不可用引用

**不可判定配置引用**：
插件因数据未就绪、网络或认证问题、或操作取消而无法确认配置是否存在的代码引用。
_Avoid_: 未解析配置引用、缺失配置

**缓存导航目标**：
由本地缓存提供、可供离线查看但不证明配置当前仍存在于 Nacos 的导航目标。
_Avoid_: 已解析配置引用、实时配置

**深度陈旧阈值**：
区分普通陈旧数据与必须强制单项刷新的时间阈值，固定为七天；它不是导航或缓存保留上限。
_Avoid_: 最大陈旧期、缓存有效期、TTL

**Namespace 索引**：
特定访问身份与 namespace 下用于全局搜索和配置存在性判断的完整配置集合。
_Avoid_: 配置详情缓存、列表页缓存

**配置摘要**：
由分页、搜索或 Namespace 索引取得的配置元数据，足以标识和定位配置，但不承诺包含正文。
_Avoid_: 配置详情、正文预览、部分配置

**配置详情**：
针对单个配置坐标独立取得的正文与完整元数据；它的取得失败不改变 Namespace 索引的完整性。
_Avoid_: 配置摘要、Namespace 索引

**配置详情缓存**：
特定访问身份、namespace、Data ID 与 Group 对应的单个配置内容副本。
_Avoid_: Namespace 索引

**历史内存缓存**：
按完整访问身份、Namespace 与配置坐标隔离的有界进程内历史列表和正文缓存；P1 不持久化它，也不承诺离线历史。
_Avoid_: 配置详情缓存、历史备份、跨身份历史

**配置坐标**：
由规范 Namespace ID、Data ID 与 Group 组成的配置唯一定位；它总与访问身份共同使用，不能脱离身份形成缓存键或发布目标。
_Avoid_: 配置摘要、请求 URL、当前选择

**主体**：
认证策略声明的稳定访问者标识，例如用户名或外部 Bearer 的用户配置别名；它参与访问身份，但不包含密码、Token 或其他秘密。
_Avoid_: 密钥、凭据快照、显示名称

**观测序号**：
远程操作开始时取得的进程内单调递增编号，用于阻止较早操作的晚到响应覆盖同一缓存坐标或访问可见性作用域上较新的观测；序号不持久化。
_Avoid_: 会话纪元、访问修订号、服务端版本

**观测高水位**：
缓存入口按配置坐标、访问身份或访问身份与 Namespace 等具体作用域记录的最新已接受观测序号；低于高水位的数据 mutation、阻断或解除阻断全部被拒绝。
_Avoid_: 观测序号、服务端版本、缓存 TTL

**列表页缓存**：
特定访问身份、namespace、搜索条件和页码对应的配置元数据页面。
_Avoid_: Namespace 索引、配置详情缓存


**数据集完整性**：
插件对预期配置是否全部取得的判断，取值为完整、部分或失败；它不表示数据是否新鲜。
_Avoid_: 数据集确认状态、缓存年龄、请求状态

**部分数据集**：
配置摘要的分页、解析或预期条目获取不完整的数据集；单个配置详情失败只改变该详情的加载状态，正文缺失只改变搜索覆盖度。
_Avoid_: 不完整缓存、失败结果

**搜索覆盖度**：
一次搜索实际检查了预期目标中的多少数据；正文搜索可为完整、已缓存详情部分覆盖或不可用，它与 Namespace 索引完整性相互独立。
_Avoid_: 数据集完整性、搜索结果数、数据集新鲜度

**诊断快照**：
用户发起连接诊断时从当前设置表单取得、仅供本次诊断使用且不改变插件活动环境的连接信息。
_Avoid_: 临时环境、未保存配置

**连接诊断**：
基于诊断快照执行的隔离、只读分阶段检查；配置与目标 namespace 可读即表示连接成功，Namespace 发现是独立的附加结果。
_Avoid_: 世代探测、测试请求、项目会话初始化
